# Список крупных задач по системам (для планирования RoadMap)

Объединяет MVP открытия депозитов (Задания 2–3) и кейс передачи ставок в кол-центры (Задание 4). Задачи сгруппированы по системам/командам и упорядочены по зависимостям.

## Команда АБС
1. Реализовать **модуль ведения и согласования ставок** в АБС (заменяет XLS), с разграничением доступа депозиты/кредиты. *(база для всего; Задание 3, UC7)*
2. **Публикация изменений ставок в Kafka** (доменные события) — единый источник для ИБ и кол-центров. *(зависит от п.1)*
3. **Интеграционный адаптер АБС** (Kafka-коннектор): приём заявок на депозит, публикация событий «ставка подтверждена», «депозит открыт».
4. Целевое: автоматизация согласования ставки (снятие ручного шага бэк-офиса).

## Команда интеграции (Kafka)
1. Развернуть/настроить **кластер Kafka**, топики ставок и заявок, схемы сообщений (schema registry).
2. Обеспечить **асинхронную развязку ИБ↔АБС** (надёжная доставка, ретраи, DLQ).

## Команда «Сервис ставок» (новая, Задание 4)
1. **Rates Consumer** — потребление событий ставок из Kafka, обновление хранилища.
2. **Хранилище актуальных ставок** (MS SQL) с отметкой времени.
3. **Rates API (REST)** для внутреннего кол-центра.
4. **File Export + Scheduler** — формирование файла ставок (CSV + манифест/checksum) по расписанию и при изменении.
5. **SFTP Connector** — доставка файла во внешнюю систему партнёра; подтверждение доставки.
6. Наблюдаемость доставки (логи, алерты, контроль свежести).

## Команда кол-центра (CRM банка)
1. Интеграция CRM с **Rates API** — отображение актуальных ставок оператору (Задание 4, UC2).
2. Организационное подключение **партнёрского кол-центра** к процессу (скрипты, маршрутизация, перевод клиента в банк).

## Команда интернет-банка
1. **Депозитный сервис** (микросервис): приём/валидация заявки, идемпотентность, БД MS SQL.
2. **Витрина/кэш ставок (read-model)** — обновляется из Kafka, мс-отклик.
3. **СМС-сервис** (силами банка): коды подтверждения и уведомления через СМС-шлюз.
4. **Web/SPA и API Gateway**: UI подачи заявки на корпоративной дизайн-системе, TLS.
5. Целевое: полный перевод ИБ на микросервисы, движение к авто-открытию депозита.

## Партнёр (внешний кол-центр)
1. Согласование **формата файла** и ключей SFTP.
2. Настройка **приёма файлов ставок по SFTP** на своей стороне.

## Инфраструктура / ЦОД
1. Среды и развёртывание новых сервисов (CI/CD), мониторинг/логирование.
2. **Резервный ЦОД, балансировка, горизонтальное масштабирование ИБ**, failover (NFR доступности 99,9%).
3. Целевое: Active-Active мультиЦОД, автоматический failover.

## Безопасность
1. TLS для сайта и ИБ; ключи и шифрование SFTP-канала.
2. Защита ПДн (152-ФЗ), контроль целостности файлов ставок, аудит доступа.
